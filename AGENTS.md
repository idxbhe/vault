# Jules AI Agent Instructions

## Git Branching Conventions
- **Naming Format:** You must use the format `[type]/[short-description]`.
- **Allowed Types:** `feat/`, `fix/`, `refactor/`, `chore/`, or `docs/`.
- **Strict Suffix Policy:** DO NOT append any random numbers, timestamps, UUIDs, or internal tracking IDs at the end of the branch name. 
- **Example of Correct Naming:** `fix/item-details` or `feat/login-page`.
- **Example of Forbidden Naming:** `fix/item-details-7421493651377904471`.
- **Constraint:** Ensure branch names are human-readable.

## Workflow Rules
- Always verify the branch name against these rules before creation.
- If a task involves multiple logical changes, prefer descriptive branch names over generic ones.
